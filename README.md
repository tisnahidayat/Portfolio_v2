# Portfolio V5 — Tisna Hidayat

Personal portfolio website built with **React + Supabase**, featuring a fully responsive public site and a protected admin dashboard.

---

## Tech Stack

| Category | Library / Tool |
|---|---|
| Frontend | ReactJS, Tailwind CSS |
| Backend | Supabase (database, storage, auth) |
| Animation | Framer Motion, AOS (Animate On Scroll) |
| UI Components | Material UI, Lucide Icons |
| Alerts | SweetAlert2 |
| Extras | canvas-confetti, axios |

---

## Features

### Visual & Interactive
- **Sparkle trail cursor** — particles follow the mouse and fade out (desktop only)
- **Scroll progress bar** — gradient bar fixed at the top of the viewport
- **3D card tilt** — project cards tilt on mouse move
- **Animated gradient text** — hero heading shifts color in a loop
- **Marquee tech stack** — two-row infinite horizontal scroll of tech logos
- **Text scramble** — navbar links scramble letters on hover
- **Magnetic buttons** — CTA buttons attract toward the cursor
- **Glowing navbar** — border-bottom glow appears when scrolled
- **Back to top** — circular progress ring button, bottom-right
- **Hire Me CTA** — floating gradient pill, bottom-left, appears after scrolling
- **Music player** — toggle button top-right; equalizer bars animate while playing
- **Like button** — heart button on each project card, saved to localStorage
- **Confetti** — bursts on successful contact form submission
- **Easter egg** — click the navbar logo 5 times quickly to trigger it

### Content Sections
- **Home** — Hero with typewriter effect, CTA buttons, social links
- **About** — Bio, animated stat counters (Supabase data), profile image
- **Resume** — Experience, Education, Organizations tabs
- **Portfolio** — Projects, Certificates, Tech Stack tabs
- **Contact** — Contact form (FormSubmit), copy-to-clipboard email, comment section

### Admin Dashboard (`/dashboard`)
- Projects — create, edit, delete; manage image, links, features, tech stack, order
- Certificates — upload and delete
- Comments — pin/unpin, delete
- Education / Experience / Organizations — full CRUD

## Getting Started

### Prerequisites
- Node.js `>= 14.x`
- npm

### 1. Clone & Install

```bash
git clone https://github.com/tisnahidayat/Portofolio_V2.git
cd Portofolio_V2
npm install --legacy-peer-deps
```

### 2. Environment Variables

Create `.env` in the project root:

```env
VITE_SUPABASE_URL=your-supabase-project-url
VITE_SUPABASE_ANON_KEY=your-supabase-anon-key
```

> Find these under **Supabase → Settings → API**.
> Never commit `.env` to version control.

### 3. Supabase Client

`src/supabase.js`:

```js
import { createClient } from '@supabase/supabase-js'

export const supabase = createClient(
  import.meta.env.VITE_SUPABASE_URL,
  import.meta.env.VITE_SUPABASE_ANON_KEY
)
```

### 4. Database Setup

Run once in **Supabase → SQL Editor**:

```sql
-- TABLES
CREATE TABLE public.projects (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title text, description text, img text, link text, github text,
  features jsonb, tech_stack jsonb,
  is_published boolean DEFAULT true,
  order_index int DEFAULT 0,
  created_at timestamptz DEFAULT now()
);

CREATE TABLE public.certificates (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  img text,
  created_at timestamptz DEFAULT now()
);

CREATE TABLE public.portfolio_comments (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  content text NOT NULL,
  user_name text NOT NULL,
  profile_image text,
  is_pinned boolean DEFAULT false,
  created_at timestamptz DEFAULT now()
);

CREATE TABLE public.profiles (
  id uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  username text UNIQUE NOT NULL,
  role text NOT NULL CHECK (role IN ('admin', 'user')),
  created_at timestamptz DEFAULT now()
);

-- RLS
ALTER TABLE public.projects ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.certificates ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.portfolio_comments ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "public read projects" ON public.projects FOR SELECT USING (true);
CREATE POLICY "public read certificates" ON public.certificates FOR SELECT USING (true);
CREATE POLICY "public read comments" ON public.portfolio_comments FOR SELECT USING (true);
CREATE POLICY "public insert comment" ON public.portfolio_comments FOR INSERT WITH CHECK (is_pinned = false);

CREATE POLICY "admin manage projects" ON public.projects FOR ALL
USING (EXISTS (SELECT 1 FROM public.profiles WHERE id = auth.uid() AND role = 'admin'));

CREATE POLICY "admin manage certificates" ON public.certificates FOR ALL
USING (EXISTS (SELECT 1 FROM public.profiles WHERE id = auth.uid() AND role = 'admin'));

CREATE POLICY "admin manage comments" ON public.portfolio_comments FOR UPDATE, DELETE
USING (EXISTS (SELECT 1 FROM public.profiles WHERE id = auth.uid() AND role = 'admin'));

-- STORAGE
INSERT INTO storage.buckets (id, name, public) VALUES ('project-images', 'project-images', true) ON CONFLICT DO NOTHING;
INSERT INTO storage.buckets (id, name, public) VALUES ('certificate-images', 'certificate-images', true) ON CONFLICT DO NOTHING;

CREATE POLICY "public read project images" ON storage.objects FOR SELECT USING (bucket_id = 'project-images');
CREATE POLICY "admin upload project images" ON storage.objects FOR INSERT
WITH CHECK (bucket_id = 'project-images' AND EXISTS (SELECT 1 FROM public.profiles WHERE id = auth.uid() AND role = 'admin'));

CREATE POLICY "public read certificate images" ON storage.objects FOR SELECT USING (bucket_id = 'certificate-images');
CREATE POLICY "admin upload certificate images" ON storage.objects FOR INSERT
WITH CHECK (bucket_id = 'certificate-images' AND EXISTS (SELECT 1 FROM public.profiles WHERE id = auth.uid() AND role = 'admin'));
```

### 5. Enable Realtime

Go to **Table Editor → portfolio_comments → Enable Realtime**.

### 6. Create Admin Account

1. **Supabase → Authentication → Users → Add User**, copy the User ID.
2. Run in SQL Editor (replace `USER_UUID`):

```sql
INSERT INTO public.profiles (id, username, role)
VALUES ('USER_UUID', 'admin', 'admin');
```

### 7. Background Music (optional)

Place your audio file at:

```
public/music.mp3
```

The music player will auto-detect it. Supported formats: `.mp3`, `.ogg`, `.wav`.

### 8. Run Locally

```bash
npm run dev
```

Open `http://localhost:5173`.

## Troubleshooting

- After changing `.env`, restart the dev server.
- If RLS blocks requests, check that the `profiles` row exists for your admin user.
- Install issues: use `npm install --legacy-peer-deps`.
- Clear browser cache if you see stale portfolio data.
